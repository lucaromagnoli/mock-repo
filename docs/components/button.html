Button Component

A flexible, accessible button component with multiple variants, sizes, and states. Supports loading states, icons, and full keyboard navigation.

Variants

Different button variants for different purposes. Use Primary for main actions, Secondary for alternative actions, Tertiary for low-emphasis actions, and Danger for destructive actions.

JavaScript 
// Primary - Main action
new Button({ text: 'Primary', variant: 'primary' });

// Secondary - Alternative action
new Button({ text: 'Secondary', variant: 'secondary' });

// Tertiary - Low-emphasis action
new Button({ text: 'Tertiary', variant: 'tertiary' });

// Danger - Destructive action
new Button({ text: 'Danger', variant: 'danger' });

Sizes

Three sizes available: Small for compact interfaces, Medium (default) for most use cases, and Large for prominent actions. All sizes meet touch target requirements.

JavaScript 
// Small (32px height, 44px on mobile)
new Button({ text: 'Small', size: 'small' });

// Medium - Default (40px height)
new Button({ text: 'Medium', size: 'medium' });

// Large (48px height - WCAG recommended)
new Button({ text: 'Large', size: 'large' });

States

Buttons support various states including default, hover, focus, active, disabled, and loading. Each state has distinct visual feedback.

JavaScript 
// Default state
new Button({ text: 'Default' });

// Disabled state
new Button({ text: 'Disabled', disabled: true });

// Loading state
new Button({ text: 'Loading', loading: true });

// Update state dynamically
button.setLoading(true);
button.setDisabled(true);

Accessibility

✓ Accessibility Features

  • Keyboard Navigation: Tab to focus, Enter/Space to activate
  • Focus Indicator: Visible focus ring with 3:1 contrast ratio
  • Screen Reader Support: Proper button semantics, aria-disabled, aria-busy for loading
  • Loading State: aria-busy="true" announced to assistive technology
  • Disabled State: aria-disabled="true" prevents interaction
  • Touch Target: Minimum 44x44px on mobile devices
  • Visual Feedback: Hover, active, and focus states don't rely on color alone
Keyboard Shortcuts 
Tab       - Move focus to button
Enter     - Activate button
Space     - Activate button
Shift+Tab - Move focus away from button
